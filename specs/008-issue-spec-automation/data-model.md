# Data Model: Issue-Driven Spec Planning Workflow

## Issue Intake Record

- **Purpose**: Represents the newly created issue that starts planning intake.
- **Attributes**:
  - Issue number
  - Issue title
  - Issue body
  - Issue URL
  - Creation timestamp
  - Eligibility decision

## Planning Handoff

- **Purpose**: Represents the reusable-workflow invocation sent to `codex-spec-kit-agent.yml`.
- **Attributes**:
  - Prompt reference
  - Command input (`$speckit.specify <issue body>`)
  - Issue body payload (normalized issue body)
  - Downstream workflow path
  - Originating issue number
  - Dispatch timestamp

## Feature Artifact Set

- **Purpose**: Represents the planning outputs created by the downstream Spec Kit workflow.
- **Attributes**:
  - Feature branch name
  - Spec directory path
  - Spec file path
  - Plan file path
  - Tasks file path
  - Checklist path
  - Originating issue number

## Run Outcome

- **Purpose**: Captures the result of one issue-intake automation attempt.
- **Attributes**:
  - Issue number
  - Outcome status (`handoff_started`, `skipped`, `failed`)
  - Outcome message
  - Duplicate detection result
  - Prompt reference
  - Visible issue comment reference
  - Completion timestamp

## Relationships

- One Issue Intake Record can produce at most one primary Planning Handoff.
- One Planning Handoff can lead to one Feature Artifact Set generated by the downstream Spec Kit workflow.
- One Issue Intake Record can have multiple Run Outcome records across retries, but only one successful primary handoff outcome.
- Each Run Outcome references exactly one Issue Intake Record and may reference one Planning Handoff and one Feature Artifact Set.
